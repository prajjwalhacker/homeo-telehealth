Core Features
1. Patient Registration (India-Specific)

Patient registration must collect detailed biodata:

Full Name

Age

Gender

Mobile Number (India – primary identifier)

Email (optional but recommended)

Address (City, State, Pincode)

Height / Weight

Blood Group

Existing medical conditions

Current medications

Lifestyle details (smoking, alcohol, etc.)

2. OTP-Based Authentication (No Passwords)
Login / Signup Flow:

User enters mobile number (India) or email

Backend generates OTP

OTP sent via SMS or email

User submits OTP

Backend verifies OTP

On success:

Create or fetch user

Issue JWT access token

User logged in

Security Requirements:

OTP expires after fixed time

OTP usable only once

Rate limiting on OTP requests

JWT used for all authenticated APIs

3. Patient Dashboard

After login, patient can:

View and update profile

See past consultation history

Start audio consultation

View saved transcripts from previous sessions

4. Audio Consultation

Patient can start an audio consultation with a doctor

Audio recorded in browser

Audio uploaded to backend

Speech converted to text asynchronously

Both audio + transcript saved and linked to patient record

5. Doctor Interaction (Basic)

Doctor can:

View patient biodata

Access audio recordings

Read converted text transcripts

(Optional) Doctor responses via text or audio
